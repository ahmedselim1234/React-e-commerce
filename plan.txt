Modernize E-Commerce Application Design & Implement Dummy Data
This document outlines the phased plan to completely redesign the React E-commerce application. The goal is to achieve a premium, state-of-the-art visual appearance with rich aesthetics (vibrant colors, glassmorphism, subtle micro-animations) and to use structured dummy data for seamless UI development.

User Review Required
IMPORTANT

Styling Approach: The project currently has both bootstrap and tailwindcss installed. To achieve the best "premium, custom modern design" as per best practices, it is often better to rely on Vanilla CSS or exclusively Tailwind rather than mixing frameworks. Are you okay with completely removing Bootstrap to ensure a consistent, modern aesthetic?

WARNING

Data Replacement: We will temporarily bypass backend API calls / Redux state fetches for displaying products and use our generated dummy data. Are there specific mock entities you want me to emphasize (e.g., specific fake product names, categories, or user scenarios)?

Proposed Changes
Phase 1: Establish Dummy Data Foundation
Create a centralized source of truth for all mock data to easily simulate the entire application state.

[NEW] src/data/dummyData.js

dummyBrands (e.g., Apple, Samsung)
